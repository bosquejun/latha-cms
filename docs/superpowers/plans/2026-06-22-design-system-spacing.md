# Design System Spacing/Typography/Layout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add layered spacing + typography tokens and wire existing layout tokens into `globals.css`, then refactor shell/card/form/view code to consume them via Tailwind v4 utilities, eliminating ad-hoc magic-number spacing.

**Architecture:** Two-layer token system in `packages/ui/src/styles/globals.css` — numeric primitive spacing scale (`--space-*`) plus intent-named semantic aliases, both exposed as Tailwind v4 utilities through `@theme inline`. A typography scale (`--text-*`) is exposed the same way. The three already-defined-but-unused layout tokens (`--sidebar-width`, `--header-height`, `--content-max`) get wired into the shell. Components are then refactored to reference semantic utilities instead of raw numbers.

**Tech Stack:** Tailwind CSS v4 (`@theme inline`), React 19, tsup (UI build), Vite (playground build/dev).

## Global Constraints

- Tailwind v4 syntax only — tokens exposed via `@theme inline`, not a JS config.
- The UI package emits no CSS itself; Tailwind utilities are generated by the **consuming app's** Vite build (playground). Verification = `pnpm --filter playground build`.
- No intended visual regressions: `72rem == max-w-6xl`, `16rem == w-64`, `3.5rem == h-14` — these specific swaps must be pixel-identical. Other swaps map to the nearest existing scale step.
- 4px rhythm — every primitive is a multiple of `0.25rem`.
- Keep standard Tailwind classes (`text-sm`, `text-xs`, `gap-6`) working; only steer to semantic names where intent is clear.
- Existing import/font/color/shadow blocks in `globals.css` must remain unchanged.

### Task 1: Add primitive + semantic spacing scale and wire layout tokens

**Files:**
- Modify: `packages/ui/src/styles/globals.css` (`:root` block ~lines 35-93, `@theme inline` block ~lines 132-180)

**Interfaces:**
- Produces: spacing utilities consumable as Tailwind classes — `p-page`, `p-sidebar`, `p-card`/`px-card`/`py-card`, `gap-section`, `gap-card`, `gap-field`, `gap-form`, `gap-inline`, `gap-stack`; layout tokens usable as `w-(--sidebar-width)`, `h-(--header-height)`, `max-w-(--content-max)`.

- [ ] **Step 1: Add the primitive + semantic spacing variables to `:root`**

Insert this block inside `:root` immediately after the `/* Layout constants */` group (after `--content-max: 72rem;`, before the closing `}` of `:root`):

```css
  /* Spacing — primitive 4px rhythm (Layer A) */
  --space-0: 0;
  --space-px: 1px;
  --space-1: 0.25rem;   /* 4px  */
  --space-2: 0.5rem;    /* 8px  */
  --space-3: 0.75rem;   /* 12px */
  --space-4: 1rem;      /* 16px */
  --space-5: 1.25rem;   /* 20px */
  --space-6: 1.5rem;    /* 24px */
  --space-8: 2rem;      /* 32px */
  --space-10: 2.5rem;   /* 40px */
  --space-12: 3rem;     /* 48px */
  --space-16: 4rem;     /* 64px */

  /* Spacing — semantic aliases (Layer B), intent-named */
  --space-page: var(--space-6);        /* page / content padding       */
  --space-section: var(--space-8);     /* gap between page sections    */
  --space-card: var(--space-6);        /* card inner padding           */
  --space-card-gap: var(--space-4);    /* gap inside a card            */
  --space-field-gap: var(--space-2);   /* label <-> control <-> help   */
  --space-form-gap: var(--space-5);    /* gap between form fields      */
  --space-inline: var(--space-2);      /* icon <-> text, button rows   */
  --space-stack: var(--space-1);       /* nav item lists               */
  --space-sidebar: var(--space-4);     /* sidebar inner padding        */
```

- [ ] **Step 2: Expose spacing + layout tokens as utilities in `@theme inline`**

Inside the `@theme inline { ... }` block, add this group just before its closing `}` (after the existing `--shadow-*` lines):

```css
  /* Spacing scale -> Tailwind spacing utilities (p-*, px-*, gap-*, m-*) */
  --spacing-page: var(--space-page);
  --spacing-section: var(--space-section);
  --spacing-card: var(--space-card);
  --spacing-card-gap: var(--space-card-gap);
  --spacing-field: var(--space-field-gap);
  --spacing-form: var(--space-form-gap);
  --spacing-inline: var(--space-inline);
  --spacing-stack: var(--space-stack);
  --spacing-sidebar: var(--space-sidebar);

  /* Layout constants -> width/height/max-width utilities */
  --spacing-sidebar-width: var(--sidebar-width);
  --spacing-header-height: var(--header-height);
  --spacing-content-max: var(--content-max);
```

Note: in Tailwind v4 a `--spacing-<name>` theme key generates `p-<name>`, `px-<name>`, `py-<name>`, `gap-<name>`, `m-<name>`, `w-<name>`, `h-<name>`, `max-w-<name>` etc. So `--spacing-card` yields `p-card`/`px-card`/`py-card`/`gap-card`; `--spacing-field` yields `gap-field`; `--spacing-content-max` yields `max-w-content-max`.

**Files:**

- [ ] **Step 1: Write `packages/ui/DESIGN.md`**

Create the file with this content:

```markdown
# Latha Design System

Tokens live in `src/styles/globals.css` and are exposed as Tailwind v4
utilities via `@theme inline`. Always prefer a semantic utility over a raw
number.

## Spacing

Layer A — primitive 4px rhythm: `--space-1` (4px) ... `--space-16` (64px).
Use these only when no semantic alias fits.

Layer B — semantic aliases (use these by default):

| Utility       | Value | Use for                          |
| ------------- | ----- | -------------------------------- |
| `p-page`      | 24px  | page / content padding           |
| `p-sidebar`   | 16px  | sidebar inner padding            |
| `p-card`      | 24px  | card inner padding               |
| `gap-section` | 32px  | between page sections            |
| `gap-card`    | 16px  | inside a card                    |
| `gap-form`    | 20px  | between form fields              |
| `gap-field`   | 8px   | label / control / helper         |
| `gap-inline`  | 8px   | icon-text, button rows           |
| `gap-stack`   | 4px   | nav item lists                   |

## Typography

| Utility        | Size / line-height | Use for                |
| -------------- | ------------------ | ---------------------- |
| `text-display` | 30 / 36            | page hero (rare)       |
| `text-h1`      | 24 / 32            | page title             |
| `text-h2`      | 20 / 28            | card / section title   |
| `text-h3`      | 16 / 24            | sub-section, topbar    |
| `text-body`    | 14 / 20            | default UI text        |
| `text-small`   | 13 / 20            | secondary, table cells |
| `text-caption` | 12 / 16            | helper, descriptions   |
| `text-label`   | 11.2 / 16          | tracked uppercase eyebrow |

Apply color (`text-muted-foreground`) and weight (`font-semibold`) on top of
the size utility as needed.

## Layout tokens

`w-(--sidebar-width)` (16rem), `h-(--header-height)` (3.5rem),
`max-w-content-max` (72rem). These are authoritative — do not hardcode
`w-64` / `h-14` / `max-w-6xl`.
```
