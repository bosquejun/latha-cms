# @kon10/client-react — React Bindings for the Delivery Client

React hooks over `@kon10/client`. A thin binding layer — all fetching logic lives in `@kon10/client`; this package only adapts it to React.

See root [`CLAUDE.md`](../../../CLAUDE.md) for global rules.

## Owns

- **Provider** — `context.tsx` (`Kon10Provider`, `useDeliveryClient`, `Kon10ProviderProps`): supplies a configured `DeliveryClient` through context.
- **Hooks** — `hooks.ts` (`useList<T>`, `useDoc<T>`, `useSingle<T>`): React data-fetching hooks over the delivery client, generic over the doc type (`JsonDoc` default).

## Must never contain

- Fetching/transport logic or the envelope contract — those belong in `@kon10/client`. This package is bindings only.

## Conventions specific to client-react

- Depends on `@kon10/client` and re-exposes its behavior through hooks; don't reimplement client logic here.
- Keep the hooks generic (`<T = JsonDoc>`) so consumers can supply types generated by `@kon10/cli`.
- Client-side only; no server imports.

## Tests

Covered indirectly via `@kon10/client` and consuming apps; no heavy unit suite here.
